# top.drluo.luo9-ai-chat

Luo9 Bot AI 对话插件 - 具备记忆、人格、情绪、主动对话的拟人化聊天系统

## 核心功能

### 记忆系统
- **工作记忆**：群聊所有消息临时存储（无论是否回复），6小时自动清理，开启群聊时自动加载
- **短期记忆**：当前会话对话历史，滑动窗口自动管理
- **长期记忆**：跨会话持久化存储，分级管理
  - 永久记忆：用户明确要求记住的信息
  - 重要记忆：AI 自动提取的姓名、喜好、经历
  - 普通记忆：闲聊内容，30天未访问自动淡忘
- **AI 记忆审查**：每小时自动整合记忆，合并重复、修正错误、删除过时
- **记忆归档**：过期和被遗忘的记忆自动归档到 `archive.json`，不会被读取但保留历史记录
- 自然语言管理：说"忘掉我刚才说的"即可清除近期记忆

### 人格系统
- 6种预设人格模板：温柔体贴、幽默风趣、理性分析、傲娇毒舌、元气活泼、安静内敛
- 6种可调特质（0.0~1.0）：幽默、温暖、好奇、正式度、详细度、共情
- 人格快照：保存/加载/切换不同人格配置
- 人格稳定性：核心特质不会在对话中突变

### 情绪系统
- 实时情绪状态：neutral/happy/sad/thinking/surprised/angry/shy/worried/tired/excited
- AI 情绪分析：每次对话后用 AI 精准分析用户情绪
- 情绪衰减：极端情绪随时间自然恢复平静
- 自然表达：情绪通过回复的语气和用词自然流露，不使用显式标签
- 互动频率影响：频繁聊天增加亲近感

### 群聊智能回复
- AI 回复决策：非@消息由 AI 综合上下文判断是否回复
- 对话跟进：@机器人后，群内短时间内可自然接话
- 主动插话权重：可调节机器人参与群聊的积极性
- 管理员控制：只有管理员可以开关对话

### 主动对话系统
- 时间问候：用户长时间未对话后发送友好问候
- 日期提醒：基于记忆中的重要日期主动祝福
- 深夜克制：夜间消息更安静
- 免打扰时段：可设置安静时间
- 识趣机制：被忽略后自动降低频率

## 快速开始

### 1. 构建
```bash
cargo build --release
```

### 2. 部署
将编译产物（`target/release/top_drluo_luo9_ai_chat.dll` 或 `.so`）放入 Luo9 Bot 插件目录。

### 3. 首次运行
启动插件后会自动生成：
```
data/plugin_ai_chat/
├── config.yaml         # 配置文件
├── prompts/
│   └── default.txt     # 默认人设提示词
├── memory.json         # 长期记忆 (运行时自动生成)
├── personality.json    # 人格配置 (运行时自动生成)
├── emotion.json        # 情绪状态 (运行时自动生成)
└── proactive.json      # 主动对话状态 (运行时自动生成)
```

### 4. 配置
编辑 `data/plugin_ai_chat/config.yaml`，**必填**以下三项：
```yaml
api_key: "sk-你的密钥"
base_url: "https://api.deepseek.com/v1"
model: "deepseek-chat"
self_qq: 123456789      # 机器人自身 QQ 号
admin_qq: 123456789    # 管理员 QQ 号
```

### 5. 重启插件

## 命令

### 对话控制 (仅管理员)

| 命令 | 说明 |
|------|------|
| `开!` / `开启对话` | 群聊：为当前群开启对话；私聊：为自己开启对话 |
| `停!` / `关闭对话` | 群聊：关闭当前群对话；私聊：关闭自己的对话 |
| `遗忘对话` | 查看并清除当前对话历史 |
| `重启对话` | 清除对话历史和所有记忆 |

### 人格管理 (仅管理员)

| 命令 | 说明 |
|------|------|
| `人格模板` | 查看可用人格模板列表 |
| `切换人格:温柔体贴` | 切换到指定人格模板 |
| `调整特质:幽默 0.8` | 调整单项特质值 (0.0~1.0) |
| `查看人格` | 查看当前人格设定详情 |
| `保存人格:备份` | 将当前人格保存为快照 |
| `加载人格:备份` | 加载已保存的人格快照 |
| ` ` | 查看所有已保存的快照 |

### 主动对话设置 (仅管理员)

| 命令 | 说明 |
|------|------|
| `开启主动对话` | 开启主动消息推送 |
| `关闭主动对话` | 关闭主动消息推送 |
| `设置免打扰:23-7` | 设置免打扰时段 (24小时制) |
| `提醒我:12-25 圣诞节` | 添加日期提醒 |

### 记忆管理 (所有用户)

在对话中自然语言管理记忆：
- "记住我喜欢喝咖啡" — 永久记忆
- "忘掉我刚才说的" — 清除近期记忆
- "忘掉xxx" — 模糊匹配删除

## 配置详解

### 顶层配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `api_key` | - | **必填** API 密钥 |
| `base_url` | - | **必填** API 地址 (OpenAI 兼容) |
| `model` | - | **必填** 模型名称 |
| `self_qq` | 0 | **必填** 机器人自身 QQ 号，用于判断群消息是否 @了机器人。设为 0 则回复所有消息 |
| `admin_qq` | 0 | 管理员 QQ 号，控制命令权限。设为 0 则所有人可用 |
| `prompts` | `default.txt` | 人设提示词文件名 (放在 prompts/ 目录下) |

### AI 调用参数 (`ai`)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `frequency_penalty` | 2.0 | 频率惩罚 (-2.0~2.0)，越高越不重复 |
| `presence_penalty` | 1.0 | 存在惩罚 (-2.0~2.0)，越高越多样 |
| `temperature` | 1.3 | 温度 (0.0~2.0)，越高越随机有创意 |
| `top_p` | 0.1 | 核采样 (0.0~1.0)，越低越集中 |
| `max_tokens` | 4096 | 主对话最大回复 token |
| `request_timeout` | 60 | 请求超时 (秒) |
| `analysis_max_tokens` | 300 | 分析任务 (记忆/情绪) 最大 token |
| `analysis_temperature` | 0.3 | 分析任务温度，越低越确定 |

### 对话参数 (`conversation`)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `max_history` | 10 | 对话历史保留轮数 (每轮 = 1条用户 + 1条AI) |
| `batch_timeout_ms` | 6000 | 消息合并超时 (毫秒)，短时间内连续消息合并为一条 |
| `typing_speed` | 5.0 | 打字模拟速度 (字符/秒)，控制分段消息的发送间隔 |
| `max_typing_delay_ms` | 4000 | 单条消息打字延迟上限 (毫秒) |
| `reply_follow_up_secs` | 300 | 对话跟进超时 (秒)，@机器人后此时间内无需再次@ |
| `intrusiveness_weight` | 0.3 | 主动插话权重 (0.0~1.0)。0.0=话痨，0.3=正常，1.0=高冷 |

### 记忆参数 (`memory`)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `normal_expire_days` | 30 | 普通记忆过期天数，0=永不过期 |
| `important_fade_days` | 7 | 重要记忆降权天数，超过后标记为[淡忘] |
| `auto_summarize_threshold` | 10 | 对话历史超过此轮数时自动摘要存为记忆 |
| `working_memory_expire_hours` | 6 | 群聊工作记忆过期时间 (小时)，所有群消息临时存储 |

### 情绪参数 (`emotion`)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `decay_rate` | 0.15 | 情绪衰减速率 (每小时)。0.1=约10小时恢复，0.3=约3小时 |
| `decay_delay_secs` | 60 | 衰减延迟 (秒)，情绪变化后此时间内不衰减 |
| `neutral_threshold` | 0.15 | 情绪强度低于此值时自动恢复为 Neutral |
| `affinity_threshold` | 3.0 | 亲近感阈值 (次/小时)，互动频率超过此值时更亲近 |

### 主动对话参数 (`proactive`)

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `enabled` | true | 是否开启主动对话 |
| `quiet_start` | 23 | 免打扰开始 (24小时制) |
| `quiet_end` | 7 | 免打扰结束 |
| `interval` | 7200 | 主动消息基础间隔 (秒)，2小时 |
| `max_ignore` | 3 | 连续忽略次数上限，达到后降低频率 |
| `low_mood_multiplier` | 2.0 | 低情绪时间隔倍率 |
| `check_interval` | 60 | 周期检查间隔 (秒) |

### 系统消息模板 (`messages`)

可自定义机器人回复的系统消息文本：
```yaml
messages:
  start:
    success: "对话已开启，你可以开始聊天了。"
    redo: "对话已开启，无需重复操作。"
  stop:
    success: "对话已停止。"
    redo: "对话未开启，无需停止。"
  forget:
    success: "已遗忘对话记录。"
    fail: "没有找到对话记录。"
  restart:
    success: "对话已重启。"
    redo: "没有找到对话记录。"
```

## 提示词 (Prompt) 编写指南

提示词文件放在 `data/plugin_ai_chat/prompts/` 目录下。

**你只需要写人设和性格**，以下技术规则由代码自动注入，无需在提示词中包含：
- 回复分隔符 (`|^|`)
- 输出格式要求
- 情绪表达规则
- 记忆融入规则
- 定时功能格式

### 提示词模板

```markdown
# 人设
你的名字叫小助手，是一个 20 岁的大学生。

# 性格
- 温柔友善，善于倾听
- 说话简短自然
- 偶尔会开玩笑

# 输出风格
- 回复简短，像朋友聊天
- 不要像机器人
```

### 参考示例

插件附带 `prompts/luo9.txt` 作为完整的角色扮演提示词参考，展示了如何定义：
- 角色背景和经历
- 外表描述
- 性格层次
- 输出风格

## 群聊行为说明

### 回复逻辑

| 消息类型 | 行为 |
|----------|------|
| @机器人 | 必定回复 |
| 群内对话跟进中 | 回复 (无需@) |
| 非@消息 | AI 综合判断是否回复 |

### AI 回复决策

当群消息没有 @机器人时，AI 会综合以下信息判断是否回复：
- 机器人的人格特质和当前情绪
- 与该用户的记忆（了解这个人是谁）
- 最近的对话上下文
- `intrusiveness_weight` 配置值

`intrusiveness_weight` 越低，机器人越容易参与无关话题；越高越克制。

### 对话跟进

用户 @机器人 发起对话后，在 `reply_follow_up_secs` 时间内：
- 该用户的后续消息无需 @
- 群里其他人的消息也会被考虑回复
- 超时后必须 @才会回复

## 调试工具

`test/debug.html` 是一个独立的调试页面，浏览器直接打开即可使用。

### 功能
- **Data Viewer**: 加载 JSON 数据文件，可视化展示记忆、工作记忆、情绪、归档数据
- **Simulator**: 模拟消息发送，查看 `decide_reply` 和 `process_message` 构建的完整上下文
- **Raw JSON**: 查看原始 JSON 数据

### 使用方法
1. 浏览器打开 `test/debug.html`
2. 将 `data/plugin_ai_chat/` 下的 JSON 文件拖入页面，或点击 "Load Test Data" 加载示例数据
3. 切换到 Simulator 标签页，输入群号/用户号/消息，点击 Simulate 查看上下文构建结果
